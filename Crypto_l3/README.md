Задание 1:
Сравнение JAX-RS и SpringREST было несколько предвзятым по причине того что на SpringREST ранее разрабатывал, на JAX-RS - нет, а в интернетах пишут что JAX-RS несколько сложнее - муторнее.
Я выбрал SpringREST в основном из соображений большей распространенности и относительной простоты разработки(плюс ранее в ходе выполнения ЛР по технологиями сетевых процессов вроде был опыт разработки на нем).
В частности, SpringREST автоматически без написания доплонительных парсеров и проверок поддерживает работу с JSON и XML.

Задание 3:
В ходе разработки было реализовано API для доступа к данным.
Для того чтобы проверить функционал можно использовать следующие запросы:
GET /api/coins-xsl  Получение валют в виде XML
GET /api/coins-json  Получение валют в виде JSON
GET /api/coins/{id}  Получение конкретной валюты
PUT /api/coins/{id}  Изменение конкретной валюты
DELETE /api/coins/{id}  Удаление конкретной валюты
Тоже самое справедливо и для бирж, для этого в запросах coins меняем на exchanges.
Можно посмотреть как это работает введя в адресную строку следующие команды:
http://localhost:8080/api/coins?format=xml Получаем список всех валют в формате XML
